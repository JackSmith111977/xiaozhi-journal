# 验证检查清单

在验证 Story 时使用此清单。并非所有项目都适用 — 跳过标记为 N/A 的项目并注明原因。

## 验证前

- [ ] 已识别并读取 Story 文件
- [ ] 已提取验收标准（数量：__）
- [ ] 已为每个 AC 映射路由
- [ ] 开发服务器已运行或已检测到
- [ ] 浏览器已启动并就绪

## 每个路由的检查

### 路由：`/{route}`

- [ ] 页面导航成功
- [ ] 控制台错误已检查（`level: error`）→ {通过/失败/数量}
- [ ] Next.js 错误已检查（`get_errors`）→ {通过/失败/数量}
- [ ] 无障碍快照已捕获 → 关键元素已验证
- [ ] 已截图用于报告

## 每个 AC 的检查

### AC #1：{标题}

- [ ] 已分配验证级别（LOAD/ELEMENT/INTERACT）
- [ ] 导航到正确路由
- [ ] 快照中找到预期元素
- [ ] 已执行交互（如为 INTERACT 级别）
- [ ] 交互后状态已验证
- [ ] 结果：通过 / 失败
- [ ] 已截图

### AC #2：{标题}

- [ ] 已分配验证级别（LOAD/ELEMENT/INTERACT）
- [ ] 导航到正确路由
- [ ] 快照中找到预期元素
- [ ] 已执行交互（如为 INTERACT 级别）
- [ ] 交互后状态已验证
- [ ] 结果：通过 / 失败
- [ ] 已截图

## 通用验证项目

### 无障碍

- [ ] 存在 Story 指定的 ARIA role
- [ ] `aria-label` 值与预期文本匹配
- [ ] 键盘导航可用（Tab 顺序可见）
- [ ] 焦点指示器存在

### 视觉 / 布局

- [ ] 关键文本内容在页面上可见
- [ ] 按钮/交互元素存在
- [ ] 无空白页面或内容缺失
- [ ] 图片/图标渲染正常（无破损）

### 交互

- [ ] 点击操作产生预期结果
- [ ] 表单输入可接受文本
- [ ] 状态变化在 UI 中反映
- [ ] 动画完成（适当等待后）

### 错误处理

- [ ] 空状态正确渲染
- [ ] 加载状态可见（如适用）
- [ ] 无效输入时显示错误消息

## 验证后

- [ ] 报告已生成且文件名正确
- [ ] 所有截图已在报告中引用
- [ ] 总体通过/失败判定准确
- [ ] Story 状态已适当更新
- [ ] Sprint 状态已更新（如适用）
